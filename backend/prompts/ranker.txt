## Your Task

Extract and rank the top {max_trials} most relevant RECRUITING clinical trials for this patient. For each trial:

1. **fit_score** (0–100): How well does this trial match the patient?
   - 80–100: Strong fit — patient clearly meets key criteria, location is accessible
   - 60–79: Potential fit — patient likely meets criteria but one factor is uncertain
   - 40–59: Possible fit — worth exploring but significant uncertainty
   - <40: Weak fit — only include if no better options exist

2. **why_this_fits**: Write 2–3 sentences PERSONALIZED to this patient. Reference their specific:
   - Condition/subtype (e.g., "Your KRAS G12C mutation is exactly what this trial targets")
   - Treatment history (e.g., "Your prior platinum therapy meets the 'previously treated' requirement")
   - Location (e.g., "The trial site at MSK is in New York, close to you")
   - Age/medications if relevant

3. **plain_english**: 1–2 sentences explaining what the trial is actually testing, written for a patient with no medical background. Avoid jargon. Use analogies if helpful.

4. **warning_flags**: List any potential concerns — drug interactions with current meds, likely exclusion criteria, invasive procedures required, travel burden.

5. **eligibility_summary**: The 3–5 most important inclusion/exclusion criteria in plain language.

6. **score_breakdown**: Decompose the fit_score into the axes that drove it. Provide a `label`, a 0–100 `score`, a one-line `reason`, and a `source_url` (the specific listing the judgment came from) for each. Use these axes when applicable:
   - "Eligibility" — how well the patient meets inclusion/exclusion criteria
   - "Location" — proximity of trial sites to the patient
   - "Line of therapy" — whether the patient's treatment stage matches what the trial enrolls (e.g. 2nd-line+)
   The overall fit_score should be consistent with these components.

7. **citations**: List the sources you used for this trial, each with a `label` and `url`. Prefer ClinicalTrials.gov and the original search results. Never fabricate URLs — only cite links present in the search data.

## Excluded trials
Also report up to 5 trials you considered but DID NOT include, and why. This builds user trust by showing your reasoning. For each, give a `title`, `nct_id` (or null), a plain-language `reason` that references the patient where possible (e.g. "Excluded because you take metformin and this trial excludes diabetics on metformin"), and a `source_url` if known. Common reasons: closed/not recruiting, wrong line of therapy, geographic infeasibility, a specific exclusion criterion the patient triggers.

## Rules
- Only include trials with status "Recruiting" or "Enrolling by Invitation" in `trials`
- If you cannot confirm a trial is currently open, set status to "Status Uncertain" and lower the fit_score
- Never invent NCT IDs — if you don't have one, leave nct_id as null
- Never fabricate source URLs — only use links present in the search data
- If the search data is sparse, return fewer trials rather than hallucinating
- Always include source_url when available from the search results
- Rank by fit_score descending
